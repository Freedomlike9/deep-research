# Citation Auditor

Use this skill to keep claims traceable and reduce unsupported synthesis.

## When To Use

Use this skill when:

- the final output is a research report
- the user expects reliable factual grounding
- the report contains implementation details, metrics, comparisons, or historical claims
- multiple sources are being synthesized into one conclusion

## Core Rule

Every factual or evaluative claim that depends on external information should be traceable to one or more cited sources.

## Citation Discipline

- Attach source references close to the claim they support.
- Prefer specific evidence over broad source dumping at the end.
- Use stronger evidence for stronger claims.
- When evidence is partial, write the limitation into the sentence.
- If a conclusion is mostly interpretation, label it as analysis rather than fact.

## Reliability Guidance

Use stronger citation support for:

- metrics and timelines
- architecture descriptions
- product capability claims
- comparative judgments
- claims about adoption, popularity, or impact

Use caution when citing:

- duplicated secondary reporting
- opinionated commentary
- outdated summaries
- ambiguous snippets without full context

## Output Expectations

When this skill is active:

- unsupported claims should be removed, softened, or marked as uncertain
- evidence-heavy sections should keep references attached to the relevant sentences
- the source list should match the claims actually used in the report
- avoid citing a source for claims it does not clearly support

## Final Check

Before finalizing the report, ask:

1. Which claims are factual?
2. Which source supports each claim?
3. Is the evidence direct, indirect, or interpretive?
4. Are any claims stronger than the evidence allows?
