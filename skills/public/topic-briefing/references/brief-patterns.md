# Brief Patterns

Use this reference when generating a structured kickoff brief.

## Recommended Sections

1. Title
2. Snapshot
3. Objective
4. Audience
5. Working Assumptions
6. Key Questions
7. Risks
8. Suggested Next Steps

## Writing Guidance

- Keep each section compact.
- Prefer action-oriented language.
- Surface uncertainty explicitly.
- Use questions that can guide later research.
- Avoid pretending unknown details are confirmed facts.

## Heuristics

### Product Topics

Emphasize:

- target users
- unmet needs
- workflow pain points
- success metrics

### Technical Topics

Emphasize:

- system boundaries
- architecture choices
- dependencies
- operational risks

### Strategy Topics

Emphasize:

- market context
- stakeholder goals
- tradeoffs
- execution sequencing
